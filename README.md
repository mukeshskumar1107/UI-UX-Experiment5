## **Aim**

To analyze user pain points, write UX problem statements, choose suitable UX research methods, and create user personas for two different application scenarios using one real website per scenario.

---

## **Algorithm**

1. Select two UX scenarios.
2. Choose one real website for each scenario.
3. Observe the UI/UX and identify user pain points.
4. Write UX problem statements using the format:
   **"[User] needs a way to [achieve goal] because [obstacle], which results in [impact]."**
5. Select appropriate UX research methods and justify choices.
6. Create personas with demographics, goals, behaviors, pain points, and motivations.
7. Add screenshots for each website.
8. Document all findings and prepare README & PDF.

## **1. User Pain Points**

* Too many products causing option overload.
* Filter panel is crowded and not immediately clear.
* Size chart details are confusing for new users.
* Checkout takes multiple unnecessary steps.

 ![1B](https://github.com/user-attachments/assets/1e872c7a-88cc-4db7-a48d-b2eed5bda8a2)

---

## **2. UX Problem Statements**

### **Problem Statement 1**

A **fashion shopper** needs a way to **find relevant products quickly** because **the filter system is cluttered**, which results in **confusion and longer decision-making time**.

### **Problem Statement 2**

A **new customer** needs a way to **understand sizing accurately** because **the size chart is unclear**, which results in **wrong purchases and product returns**.

---

## **3. Research Methods & Justification**

| Method                | Why Selected                                                |
| --------------------- | ----------------------------------------------------------- |
| **Usability Testing** | Reveals how users navigate filters, products, and checkout. |
| **Surveys**           | Helps gather large-scale feedback on shopping frustrations. |

**Justification:**
E-commerce shopping involves task-based interactions.
Usability tests capture **interaction issues**, while surveys gather **user expectations**.

## **1. User Pain Points**

* Dashboard is cluttered for beginners.
* Logging food requires multiple manual steps.
* Too many notifications/reminders.
* Workout routines lack simple beginner-friendly guides.

![2B](https://github.com/user-attachments/assets/dae96cd7-c7ce-4f92-8631-e03f8394d379)

---

## **2. UX Problem Statements**

### **Problem Statement 1**

A **beginner fitness user** needs a way to **navigate the dashboard easily** because **too many features are shown at once**, which results in **confusion and irregular usage**.

### **Problem Statement 2**

A **health-conscious user** needs a way to **get personalized diet suggestions** because **the current plans feel generic**, which results in **low motivation and poor engagement**.

---

## **3. Research Methods & Justification**

| Method            | Why Selected                                             |
| ----------------- | -------------------------------------------------------- |
| **Interviews**    | Helps understand emotional motivations and health goals. |
| **Diary Studies** | Shows daily usage patterns, habit-building issues.       |

**Justification:**
Fitness tracking is a **daily routine**, so diary studies reveal long-term behavior.
Interviews help uncover **personal goals and emotional barriers**.
